🔍 Key Research Findings

liboqs is fastest for NIST standards (35% of wins)
AVX2 hardware-optimized assembly makes ML-KEM keygen 3.3× faster than PQClean reference and 10× faster than pqcrypto. This is the library to use in production.
CIRCL (Go) wins ML-KEM encaps/decaps
Cloudflare's Go implementation achieves 94,000 encaps/sec for ML-KEM-512 — beating liboqs (61k). Used in real Cloudflare TLS deployments.
Falcon signing: 17× difference between libraries
liboqs Falcon-512 sign: 0.31ms. pqcrypto: 5.28ms. Same algorithm, 17× gap — entirely due to AVX2 NTRU sampler vs portable C code.
Java (Bouncy Castle) is 10–50× slower
Falcon-512 keygen: liboqs 7.8ms, Bouncy Castle 47.4ms. JVM overhead is unavoidable. Only use if you need the JVM ecosystem or FIPS certification.
McEliece: keygen is impractical, key sizes massive
McEliece keygen takes 129–166ms with a 261KB public key. However, encapsulation is only 0.07ms. The key size is the real barrier — not speed.
